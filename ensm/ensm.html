Contexte

Le School L@B est la cellule informatique de l'ENSM-BPN chargée de développer et maintenir les logiciels pédagogiques de l'école. La Cellule SI travaille notamment sur un logiciel de suivi pédagogique des élèves, dont la refonte web (Symfony + Vue.js) est en cours.

Ma mission consistait à créer un outil complémentaire connecté à l'API du logiciel précédent, pour la visualisation numérique du Livret ATO — carnet de formation physique des atomiciens, actuellement envoyé par courrier entre les écoles.

Stack technique

Vue.js + TypeScript Symfony (PHP) MariaDB TailwindCSS PrimeVue Pinia Vite HeidiSQL API REST

Missions réalisées

  • Développement du front-end Vue.js + TypeScript : composants réutilisables, navigation, affichage PDF via <iframe>
  • Création d'un système de rendu dynamique basé sur deux JSON (model + data) via un composant récursif
  • Développement des routes API Symfony pour la récupération des utilisateurs filtrés par grade
  • Écriture des requêtes SQL pour la BDD MariaDB et intégration dans les controllers
  • Intégration de l'API dans le front : récupération des livrets, des appréciations et des données personnelles au chargement

Contraintes particulières

Le développement s'est fait dans un environnement isolé du réseau Internet, requis pour l'homologation du logiciel militaire. L'installation de dépendances npm était impossible en ligne : chaque bibliothèque devait être téléchargée manuellement sur un poste connecté, vérifiée via un protocole d'intégrité, puis transférée vers un dépôt npm local.
