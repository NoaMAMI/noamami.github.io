Fév. – Avr. 2024 Projet BTS CIEL 1 IoT · LoRaWAN

Ballon Sonde — Station météo connectée

Lycée de l'Espace (Pierre-Paul Riquet) · Projet « Un ballon pour l'école » en partenariat avec le CNES

Contexte

Dans le cadre du projet « Un ballon pour l'école », les élèves de STI2D du lycée développent un ballon sonde capable de relever des données environnementales (température, pression, humidité, altitude…) dans l'atmosphère. Notre équipe de 4 techniciens informatiques (BTS CIEL) était chargée de concevoir et réaliser toute la chaîne numérique : de l'acquisition des mesures à bord de la nacelle jusqu'à leur visualisation sur un site web public.

Le projet s'est déroulé en deux prototypes : un premier démonstrateur lâché à quelques dizaines de mètres dans l'enceinte du lycée, puis un ballon opérationnel destiné à monter à ~1000 m d'altitude et à transmettre ses mesures à distance pendant plusieurs mois.

La chaîne IoT de bout en bout

1

Acquisition · nacelle

Capteurs environnementaux (température, humidité, pression) lus en I2C par le microcontrôleur (Nucléo / ESP32). Calcul de l'altitude à partir de la pression.

2

Transmission · LoRaWAN

Données encodées au format CayenneLPP puis envoyées par le module LoRa-E5 (piloté en commandes AT via RS232) vers une passerelle LoRaWAN.

3

Network Server · TTN

The Things Network reçoit les trames, les décode en JSON via le payload formatter et les expose sur un broker MQTT.

4

Application Server · Node-RED

Un Raspberry Pi (Node-RED + broker MQTT Mosquitto) s'abonne aux données, les traite et les enregistre, puis les transfère par FTP vers l'hébergeur.

5

Stockage · Base de données

Les mesures sont historisées dans une base MySQL hébergée sur Alwaysdata (modélisation MCD, requêtes SQL, intégration Python).

6

Visualisation · Dashboard web

Tableau de bord dynamique (jQuery / jQWidgets) affichant la température, l'humidité et la pression sur des jauges mises à jour avec les dernières données reçues.

Stack technique

C++ / mbed OS LoRaWAN The Things Network Node-RED MQTT (Mosquitto) MySQL ESP32 / Nucléo Module LoRa-E5 Capteurs I2C (HTS221, LPS22HB) CayenneLPP Commandes AT / RS232 Raspberry Pi Python HTML / CSS / Bootstrap jQuery / jQWidgets Apache · Alwaysdata Git

Travail réalisé

  • Programmation de l'acquisition des capteurs en C++ / mbed OS (bus I2C, classes HTS221Sensor et LPS22HBSensor) et calcul de l'altitude par la formule barométrique
  • Encodage des mesures au format CayenneLPP pour réduire la taille des trames sur le réseau LoRaWAN
  • Configuration et pilotage du module LoRa-E5 en commandes AT (DevEUI, AppKey, connexion OTAA) via la liaison RS232
  • Mise en place de l'application et d'un device sur The Things Network, configuration du payload formatter
  • Création d'un flux Node-RED (node MQTT + clé API) pour recevoir, traiter et enregistrer les données
  • Conception de la base de données (MCD, jointures) et enregistrement des mesures en SQL / Python
  • Réalisation du site web : IHM statique (Bootstrap) puis tableau de bord dynamique avec jauges jQWidgets

Prise en compte de la cybersécurité

Le cahier des charges imposait de traiter le risque cyber à deux niveaux : la sécurité des données et le contrôle d'accès aux équipements. Nous avons réalisé une analyse des risques de l'infrastructure réseau et appliqué des mesures concrètes, comme l'usage de clés API en lecture seule pour la récupération des données depuis TTN, afin de limiter la surface d'attaque.
